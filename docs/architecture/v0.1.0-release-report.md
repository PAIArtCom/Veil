## Release Judgment

OpenCloak OSS v0.1.0 is locally hardened and release-cut materials are refreshed after the
post-review blocker repair checkpoint, current token-prefix change, provider request
batch-apply optimization, final systematic RC rereview, and final pre-publication masking
leak repair. Claude Code and the local Codex CLI Responses path have passed controlled
live runs after the latest provider wire changes;
provider-bound payloads contained `OpenCloak_` tokens and no throwaway plaintext values. A
follow-up release/security/performance rereview found no release blockers. The repository
should not claim a separate official `https://api.openai.com` service end-to-end acceptance
run, but that separate run is not required for the v0.1.0 OpenAI Responses protocol claim.

The bounded claim that is currently supported:

- Claude Code / Anthropic Messages proxy path is live-accepted from Phase 0 and remains
  regression-covered for provider-native text fields, tool-use arguments, tool-result
  text, and streaming text/tool deltas.
- OpenAI Responses provider path for Codex is implemented, offline-verified, and
  live-accepted for the local Codex CLI Responses path, including request masking, buffered
  restore, streaming restore, tool-call arguments, tool results, prompt string variables,
  unsupported-shape fail-closed behavior, and proxy route-level streaming coverage. This is
  the v0.1.0 OpenAI Responses protocol evidence.
- Public SDK embed contract is validated through the maintained `examples/embed` reference
  integration.
- Local JSON policy files support `token`, `ignore`, and `block` with strict fail-closed
  validation, including rejection of policies that ignore every supported sensitive type.
- L1 detector suppressors are scoped so high-confidence provider-prefix secrets are not
  dropped by generic hash/business-id suppressors, while code-reference false positives are
  suppressed on the generic and entropy paths.
- Unsupported proxy endpoints fail closed before provider egress.
- The current token namespace is `OpenCloak_<TYPE>_<id>` per ADR-0014, with post-change
  Claude and Codex live acceptance refreshed on 2026-06-20.
- Provider request masking now skips unchanged spans and applies changed Anthropic/OpenAI
  Responses JSON string spans with a one-pass range rewrite when ranges are available,
  while preserving structural `sjson` fallback behavior.
- Hex-only credential values in strong `api_key`/`apikey`/`token` contexts are masked as
  secrets, and known `OpenCloak_` token prefixes remain restorable when immediately
  followed by extra lowercase hex. A later outbound mask pass treats that extra hex suffix
  as new `SECRET` text instead of hiding it inside the token idempotency guard.

Out-of-scope and non-claims:

- A separate direct `https://api.openai.com` official-service end-to-end live run is not
  claimed and is not part of the v0.1.0 release gate. A historical 2026-06-17 probe with
  the built-in OpenAI provider found that the local OpenAI API key was invalid; that probe
  is retained only as historical context.
- OCR, document parsing, attachment rewriting, or regenerated media/document payloads.
  Anthropic image/document blocks are opaque provider payloads and are not part of the
  v0.1.0 de-identification surface.
- Provider thinking/control traces as user text. They preserve provider-native semantics
  and are not masked as prompt or tool payloads.

## Known Limits

- Codex/OpenAI Responses is live-accepted for the local Codex CLI Responses path, which is
  the v0.1.0 OpenAI Responses protocol evidence. A separate direct
  `https://api.openai.com` official-service run is not part of the release gate and is not
  claimed.
- The protected provider surface is text and agentic tool I/O. Opaque media/document
  attachments and provider thinking/control traces pass through under provider-native
  semantics and are not claimed as de-identified.
- OpenAI Chat, Gemini, remote MCP egress classification, HTTP/gRPC service, local web
  console, L2 default-on semantic PII, `redact`, `format_preserving`, and configurable rule
  packs are planned or reserved.
- v0.1.0 is not a v1.0 API freeze.
- The proxy is a local loopback tool for one-person use, not a multi-tenant network service.
