## Release Judgment

OpenCloak OSS v0.1.0 is locally hardened and release-cut materials are refreshed after the
post-review blocker repair checkpoint. The local Codex CLI Responses path has passed a
controlled live run through a Responses-compatible upstream. The repository should not
claim direct `https://api.openai.com` upstream live acceptance because the local OpenAI API
key was rejected by the API during the redacted probe.

The bounded claim that is currently supported:

- Claude Code / Anthropic Messages proxy path is live-accepted from Phase 0 and remains
  regression-covered.
- OpenAI Responses provider path for Codex is implemented, offline-verified, and
  live-accepted for the local Codex CLI Responses path, including request masking, buffered
  restore, streaming restore, tool-call arguments, tool results, prompt string variables,
  unsupported-shape fail-closed behavior, and proxy route-level streaming coverage.
- Public SDK embed contract is validated through the maintained `examples/embed` reference
  integration.
- Local JSON policy files support `token`, `ignore`, and `block` with strict fail-closed
  validation, including rejection of policies that ignore every supported sensitive type.
- L1 detector suppressors are scoped so high-confidence provider-prefix secrets are not
  dropped by generic hash/business-id suppressors, while code-reference false positives are
  suppressed on the generic and entropy paths.
- Unsupported proxy endpoints fail closed before provider egress.

The claim that is not yet supported:

- Direct `https://api.openai.com` upstream live acceptance. The 2026-06-17 run used a
  local Responses-compatible upstream because the local OpenAI API key was invalid.

## Known Limits

- Codex/OpenAI Responses is live-accepted for the local Codex CLI Responses path. Direct
  `https://api.openai.com` upstream acceptance is not claimed until a valid OpenAI API key
  is available.
- OpenAI Chat, Gemini, remote MCP egress classification, HTTP/gRPC service, local web
  console, L2 default-on semantic PII, `redact`, `format_preserving`, and configurable rule
  packs are planned or reserved.
- v0.1.0 is not a v1.0 API freeze.
- The proxy is a local loopback tool for one-person use, not a multi-tenant network service.
